Practicas GIT



  1. git --version (Indica la versión instalada de Git)
  2. git init (Inicia un repositorio en el directorio donde se ejecuta el comando)
  3. git status (Inspeccionar repositorio Git)
  4. git add . (Añade todos los archivos que se encuentran dentro del directorio)
  5. git add "nombre del archivo" (Añade el archivo especificado al repositorio de Git)
  6. git rm --cached "nombre del archivo" (Deja de monitorear o de hacer el seguimiento a dicho archivo)
  7. git commit -m "comentario" (Permite especificar, a modo de comentario, los cambios realizados en el arhivo o proyecto)
  8. git config --global user.name "nombre del usuario que está realizando los cambios" (Configura en GIT el nombre para todos los proyectos que se relizarán)
  9. git config --global user.email "email del usuario que está realizando los cambios" (Configura en GIT el mail para todos los proyectos que se relizarán)
  10. git config --local user.name "nombre del usuario que está realizando los cambios" (Solo para el proyeto actual)
  11. git config --local user.email "email del usuario que está realizando los cambios" (Solo para el proyecto actual)
  12. git log (Visualiza el historial de commits en varias líneas)
  13. git log --oneline (Visualiza el historial de commits en una sola línea)
  14. git log -p (Visualiza el historial de commits junto con los detalles de los cambios realizados en el código. Para salir de los detalles escribir: q)
  15. git config user.name (Devuelve el nombre de la persona que realizó el commit)
  16. git config user.email (Devuelve el correo de la persona que realizó el commit)
  17. git init --bare (Crea un repositorio que no tendrá la working tree, o sea, no contendrá una copia de nuestros archivos. Como el repositorio servirá solo como servidor, para que otros miembros del equipo sincronicen sus trabajos, disminuimos espacio de almacenamiento de esta forma)
  18. git remote (Muestra un listado de los repositorios remotos)
  19. git remote add "escribir un nombre para el servidor local" "dirección del servidor remoto" (Permite agregar un servidor remoto a un servidor local. De esta forma tendremos un link de nuestro repositorio local con el repositorio remoto)
  20. git remote -v (Indica si el servidor local está apuntando hacia el servidor remoto)
  21. git clone "dirección del repositorio" "nombre que deseo colocar al repositorio clonado" (Permite clonar un repositorio)
  22. git push "nombre del servidor" "nombre del branch" (Publica nuestro branch del proyecto en el servidor local)
  23. git remote rename "nombre actual del repositorio remoto" "nombre por el que se desea cambiar" (Permite cambiar el nombre de un repositorio)
  24. git pull "nombre del servidor" "nombre del branch" (Copia el branch del proyecto publicado en el servidor)
  25. git branch (Lista todas las branches o ramas del proyecto)
  26. git branch "nombre del branch" (Permite crear una branch o rama del proyecto)
  27. git checkout "nombre del branch" (Nos lleva hacia la branch o rama especificada)
  28. git checkout -b "nombre del branch" (Crea una branch y nos lleva hacia ella)
  29. git merge "nombre del branch" (Agrega los cambios realizados en el branch especificado a la rama del proyecto desde donde se ejecuta el comando)
  30. git rebase "nombre del branch" (Agrega los cambios realizados en el branch especificado a la rama del proyecto desde donde se ejecuta el comando. Los commits de los cambios realizados en el branch se escribirán antes del último commit realizado en el proyecto. Este es un proceso lineal de merge)
  31. git log --graph (Permite visualizar de forma gráfica los commits realizados)
  32. git restore "nombre del archivo" (Deshace una modificación que aún no fue agregada al index o stage, o sea, antes de hacer git add)
  33. git restore --staged "nombre del archivo" (Deshace una modificación después de agregarla, pero antes de commitearla)
  34. git revert "id del commit al cual queremos regresar" (Deshace una modificación después de realizar el commit)
  35. git stash (Almacena temporalmente las modificaciones que se realice al proyecto)
  36. git stash list (Permite visualizar una lista de los cambios realizados en el stash)
  37. git stash drop "número" (Elimina un elemento determinado del stash)
  38. git stash pop (Aplica y elimina el último cambio que se agregó al stash)
  39. git diff (Permite visualizar las modificaciones realizadas en determinado código)
  40. git diff "hash inicio".."hash de fin" (Muestra los cambios realizados desde el commit de inicio al commit de fin)
  41. git tag -a "nombre de la etiqueta ejemplo: v0.1.0" (Genera una release del programa)
